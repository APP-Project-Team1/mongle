# 🚀 PRD: 기능 1 - 일정 관리 (월 캘린더 및 타임라인)

## 👥 1. 사용자 스토리 (User Stories)

> 사용자 입장에서 기능의 가치를 정의합니다.

- **As a** 바쁜 업무와 결혼 준비를 병행하는 30대 예비신부 '지아'와 담당 플래너 '수진',
- **I want to** 월 캘린더와 타임라인 뷰를 통해 결혼 준비 단계를 실시간으로 공유하고 확인하며,
- **So that** 결혼 준비의 전체 흐름을 놓치지 않고 체계적으로 관리할 수 있다. (Must / P1)

---

## 🛠 2. 상세 기능 요구사항 (Functional Requirements)

- [ ] **기능 A (일정 등록 및 수정):** 플래너 또는 커플이 주요 일정(드레스 투어, 스튜디오 촬영 등)을 캘린더에 등록 및 수정할 수 있어야 함.
- [ ] **기능 B (타임라인 뷰 제공):** 디데이(D-Day)를 기준으로 앞으로 남은 준비 단계를 타임라인 형태로 시각화하여 제공해야 함.
- [ ] **예외 케이스 처리:** 네트워크 지연 시 "일정을 불러오는 중입니다" 로딩 스피너 노출. 동시 수정 발생 시 마지막 저장 기준 덮어쓰기 및 안내 알림 제공.

## ✅ 4. 수락 기준 (Acceptance Criteria)

> 기능이 완료되었다고 판단하기 위한 최소 조건입니다.

- [ ] 사용자가 새로운 일정을 등록했을 때, 즉시 캘린더와 타임라인에 반영되어야 한다.
- [ ] 플래너가 일정을 추가/변경하면 연결된 커플 계정에서도 새로고침 없이(또는 진입 시) 해당 내용을 확인할 수 있어야 한다.
- [ ] 다른 기능(예산 관리, 리마인드 등)과 연동 시 해당 날짜 데이터가 정상적으로 전달되어야 한다.

---

## ⚙️ 5. 기술적 고려사항 및 제약 (Non-functional / Tech Notes)

- **API 연동:** 일정 CRUD 처리를 위한 REST API 엔드포인트 필요.
- **성능/보안:** 플래너와 연결된 특정 커플 계정만 해당 일정을 열람/수정할 수 있도록 데이터 접근 권한(Authorization) 체크 필수.
- **의존성:** 기능 5(로그인 및 역할별 연결)가 먼저 구현되어 있어야 데이터 매핑이 가능함.
