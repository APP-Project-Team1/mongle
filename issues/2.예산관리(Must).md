# 🚀 PRD: 기능 2 - 예산 관리 (비용 명세 및 진행 상태)

## 👥 1. 사용자 스토리 (User Stories)

> 사용자 입장에서 기능의 가치를 정의합니다.

- **As a** 투명한 비용 관리와 가성비 방어가 중요한 예비신랑 '현우',
- **I want to** 항목별 비용 명세를 실시간으로 확인하고 예산 대비 진행 바를 시각적으로 파악하며,
- **So that** 불투명한 웨딩 비용을 명확히 파악하고 예산 초과를 방지할 수 있다. (Must / P1)

---

## 🛠 2. 상세 기능 요구사항 (Functional Requirements)

- [ ] **기능 A (항목별 비용 입력):** 스·드·메, 홀, 예물 등 카테고리별로 총액, 계약금, 잔금을 입력하고 저장할 수 있어야 함.
- [ ] **기능 B (진행 바 시각화):** 설정한 총예산 대비 현재 지출 및 예정 금액을 %와 진행 바(Progress Bar) 형태로 상단에 표시.
- [ ] **예외 케이스 처리:** 예산 초과액 입력 시 진행 바 색상이 붉은색으로 변경되며 "예산을 초과했습니다" 경고 문구 노출. 입력값이 숫자가 아닐 경우 폼 검증 에러 반환.

## ✅ 4. 수락 기준 (Acceptance Criteria)

## ⚙️ 5. 기술적 고려사항 및 제약 (Non-functional / Tech Notes)

- **API 연동:** 예산 카테고리별 합산 및 진행률 계산을 처리하는 API 필요 (프론트/백엔드 교차 검증).
- **성능/보안:** 민감한 금융/자산 정보이므로 전송 구간 암호화(HTTPS) 및 타인 노출 절대 방지.
- **의존성:** 일정 관리(기능 1)의 결제일 데이터와 연동 필수.
