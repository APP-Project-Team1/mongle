# 🚀 PRD: 기능 3 - AI 챗봇 (웨딩 도메인 특화 LLM)

## 👥 1. 사용자 스토리 (User Stories)

> 사용자 입장에서 기능의 가치를 정의합니다.

- **As a** 결혼 준비 용어가 낯설고 수많은 정보 속에서 길을 잃은 예비신부 '지아',
- **I want to** 웨딩 도메인에 특화된 AI 챗봇에게 궁금한 점을 묻고 즉시 답변을 받으며,
- **So that** 정보 검색 시간을 단축하고 내 일정과 취향에 맞는 업체를 빠르고 정확하게 추천받을 수 있다. (Must / P1)

---

## 🛠 2. 상세 기능 요구사항 (Functional Requirements)

- [ ] **기능 A (자연어 질의응답):** 사용자의 웨딩 관련 질문(예: "본식 스냅 언제 예약해야 해?")에 대해 실시간으로 적절한 답변 생성 및 제공.
- [ ] **기능 B (맞춤형 정보 추천):** 사용자의 예산, 등록된 일정 데이터를 바탕으로 업체나 남은 준비 일정을 챗봇 인터페이스 내에서 추천.
- [ ] **예외 케이스 처리:** LLM 응답 지연 시 "AI가 답변을 고민하고 있어요" 애니메이션 표시. 할루시네이션(거짓 정보) 방지를 위해 정보 출처 링크 제공 및 책임 한계 고지 문구 노출.

## ✅ 4. 수락 기준 (Acceptance Criteria)

> 기능이 완료되었다고 판단하기 위한 최소 조건입니다.

- [ ] 사용자가 메시지를 전송한 후 최대 3~5초 이내에 스트리밍 형태로 답변이 출력되기 시작해야 한다.
- [ ] 챗봇이 추천한 업체 카드를 클릭하면, 해당 업체의 상세 페이지(기능 4)로 정상적으로 이동해야 한다.
- [ ] 서비스 내 보유한 웨딩 데이터베이스를 기반으로 답변하는 프롬프트 엔지니어링 또는 RAG 구축이 완료되어야 한다.

## ⚙️ 5. 기술적 고려사항 및 제약 (Non-functional / Tech Notes)

- **API 연동:** OpenAI, Anthropic 등 외부 LLM API 또는 자체 구축 모델 엔드포인트 연동.
- **성능/보안:** API 호출 비용 관리를 위해 유저당 일일 질문 횟수 제한(Rate Limiting) 로직 필요.
- **의존성:** 내부 업체 데이터베이스(기능 4) 구축 및 AI 연동을 위한 데이터 파이프라인.
