# 🚀 PRD: 기능 5 - 로그인 및 계정 연결 (커플/플래너)

## 👥 1. 사용자 스토리 (User Stories)

> 사용자 입장에서 기능의 가치를 정의합니다.

- **As a** 각자의 역할에 맞게 앱에 접속해 파트너 및 플래너 '수진'과 진행 상황을 실시간으로 연동하려는 예비부부,
- **I want to** 나의 역할에 맞게 로그인하고 초대 코드를 통해 서로의 계정을 연결하며,
- **So that** 안전하게 내 정보를 관리하고 준비 과정을 관련자들과 실시간으로 공유할 수 있다. (Should / P2)

---

## 🛠 2. 상세 기능 요구사항 (Functional Requirements)

- [ ] **기능 A (역할별 로그인/가입):** 카카오/네이버/애플 소셜 로그인 제공, 가입 시 '커플' 또는 '플래너' 역할 선택.
- [ ] **기능 B (초대 및 연결):** 고유 초대 코드를 생성하여 카카오톡 등으로 공유하고, 코드를 입력한 상대방과 계정(데이터)을 연동.
- [ ] **예외 케이스 처리:** 유효하지 않거나 만료된 초대 코드 입력 시 "잘못된 코드입니다" 알림 제공. 이미 연결된 상대가 있을 경우 중복 연결 방지.

## ✅ 4. 수락 기준 (Acceptance Criteria)

> 기능이 완료되었다고 판단하기 위한 최소 조건입니다.

- [ ] 역할(커플/플래너)을 선택하고 소셜 로그인을 완료하면 해당 역할에 맞는 메인 화면으로 라우팅되어야 한다.
- [ ] 초대 코드를 입력하여 연결이 완료되면, 즉시 두 계정 간의 일정(기능 1)과 예산(기능 2) 데이터가 동기화되어야 한다.
- [ ] 연결 해제 기능을 통해 언제든 플래너와의 연동을 취소할 수 있어야 한다.

## ⚙️ 5. 기술적 고려사항 및 제약 (Non-functional / Tech Notes)

- **API 연동:** OAuth 2.0 소셜 로그인 연동, 초대 코드 생성 및 유효성 검증 API.
- **성능/보안:** JWT 토큰 기반 인증. 초대 코드는 추측 불가능한 난수로 생성하고 일정 시간 후 만료(TTL) 처리.
- **의존성:** 모든 기능의 베이스 인프라이므로 가장 먼저 백엔드 구조가 잡혀야 함.
