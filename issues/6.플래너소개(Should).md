# 🚀 PRD: 기능 6 - 플래너 소개 (프로필 및 상담 신청)

## 👥 1. 사용자 스토리 (User Stories)

> 사용자 입장에서 기능의 가치를 정의합니다.

- **As a** 광고가 아닌 실제 평점과 포트폴리오를 보고 나와 결이 맞는 플래너를 찾고 싶은 예비신부 '지아',
- **I want to** 활동 중인 플래너의 경력, 평점, 스타일 태그를 확인하고 상담을 신청하며,
- **So that** 투명하게 공개된 정보를 바탕으로 나와 가장 잘 맞는 플래너를 안심하고 섭외할 수 있다. (Should / P2)

---

## 🛠 2. 상세 기능 요구사항 (Functional Requirements)

- [ ] **기능 A (플래너 리스트 및 프로필):** 플래너 카드 그리드 뷰 제공. 상세 페이지에서 소속, 경력, 실제 고객 평점, 주력 스타일 태그 공개.
- [ ] **기능 B (상담 신청):** 마음에 드는 플래너에게 원하는 날짜와 시간, 기본 예산 대역을 적어 상담을 신청(In-app 메시지 또는 알림 전송).
- [ ] **예외 케이스 처리:** 플래너가 상담 불가능한 상태(휴가, 스케줄 마감)일 경우 "현재 상담 접수 마감" 딤(Dim) 처리 및 버튼 비활성화.

## ✅ 4. 수락 기준 (Acceptance Criteria)

> 기능이 완료되었다고 판단하기 위한 최소 조건입니다.

- [ ] 스타일 태그(예: #꼼꼼함, #가성비, #프리미엄)를 클릭하면 해당 태그를 가진 플래너만 필터링되어 나타나야 한다.
- [ ] 상담 신청 폼 제출 시, 해당 플래너 계정으로 푸시 알림과 함께 상담 요청 내역이 성공적으로 전달되어야 한다.
- [ ] 플래너 프로필 내 평점은 조작 방지를 위해 '실제 연결되어 일정을 마친 커플'만 작성한 데이터로 계산되어야 한다.

## ⚙️ 5. 기술적 고려사항 및 제약 (Non-functional / Tech Notes)

- **API 연동:** 플래너 프로필 CRUD API, 상담 신청 내역 전송(Notification) API.
- **성능/보안:** 개인정보(연락처 등) 마스킹 처리 후 매칭 성사 시에만 공개되도록 접근 제어.
- **의존성:** 업체 정보(기능 4)와 플래너가 자주 거래하는 제휴 업체 정보 간의 연결망 구축.
