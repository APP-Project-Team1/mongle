# 🚀 PRD: 기능 8 - 리마인드 (일정 및 잔금 알림)

## 👥 1. 사용자 스토리 (User Stories)

> 사용자 입장에서 기능의 가치를 정의합니다.

- **As a** 일상에 치여 수십 개의 잔금 결제일과 예약 일정을 기억하기 벅찬 예비신부 '지아',
- **I want to** 주요 일정 및 업체 잔금 결제일에 대해 D-7, D-3, D-1 간격으로 자동 알림을 받으며,
- **So that** 바쁜 일상 속에서도 중요한 일정과 금전적 결제를 실수로 누락하지 않을 수 있다. (Should / P2)

---

## 🛠 2. 상세 기능 요구사항 (Functional Requirements)

- [ ] **기능 A (자동 푸시 알림):** 등록된 일정 및 잔금 결제일을 기준으로 D-7, D-3, D-1 정해진 시간(예: 오전 10시)에 푸시/앱 내 알림 발송.
- [ ] **기능 B (알림 센터):** 발송된 리마인드 메시지를 모아볼 수 있는 종 모양의 알림 센터(Notification Center) UI 제공.
- [ ] **예외 케이스 처리:** 사용자가 시스템 푸시 알림을 껐을 경우, 앱 진입 시 홈 화면 최상단에 배너 형태로 중요 리마인드 노출.

## ✅ 4. 수락 기준 (Acceptance Criteria)

> 기능이 완료되었다고 판단하기 위한 최소 조건입니다.

- [ ] 예산 관리(기능 2)에서 잔금 '결제 완료'를 체크하면, 예정되어 있던 해당 건의 푸시 알림 발송이 즉시 취소되어야 한다.
- [ ] 알림 발송 서버에 장애가 나더라도 중복 알림이 전송되지 않도록 멱등성(Idempotency)이 보장되어야 한다.
- [ ] 알림을 터치하면 앱이 실행되며 해당 일정이나 예산 상세 페이지로 정확히 이동(Deep-linking)해야 한다.

---

## ⚙️ 5. 기술적 고려사항 및 제약 (Non-functional / Tech Notes)

- **API 연동:** FCM(Firebase Cloud Messaging) 또는 APNs 연동을 통한 푸시 발송 서버.
- **성능/보안:** 매일 정해진 시간에 대규모 발송이 일어나는 Batch/Cron Job 스케줄러 설계 필요.
- **의존성:** 일정 관리(기능 1), 예산 관리(기능 2)의 Date 데이터 포맷 일치.
