# 🚀 PRD: 기능 9 - 견적 분석·문서화 (PDF 다운로드 등)

## 👥 1. 사용자 스토리 (User Stories)

> 사용자 입장에서 기능의 가치를 정의합니다.

- **As a** 여러 업체의 복잡한 견적을 표로 한눈에 비교하고, 부모님이나 파트너에게 깔끔한 문서로 공유하고 싶은 예비신랑 '현우',
- **I want to** 여러 업체의 견적을 한눈에 비교할 수 있는 뷰를 보고 최종 견적서를 PDF로 다운로드하며,
- **So that** 견적 내용을 객관적으로 분석하고 가족이나 다른 사람과 쉽게 공유/보관할 수 있다. (Could / P3)

---

## 🛠 2. 상세 기능 요구사항 (Functional Requirements)

- [ ] **기능 A (견적 비교 뷰):** 장바구니에 담듯 2~3개의 업체를 선택해 항목별(가격, 구성품, 옵션) 차이를 표 형태로 비교하는 기능.
- [ ] **기능 B (문서 추출):** 최종 확정된 내 예산안 및 견적서 전체를 깔끔한 템플릿의 PDF 파일로 생성 및 로컬 다운로드/공유.
- [ ] **예외 케이스 처리:** 데이터가 렌더링되기 전에 PDF 다운로드 요청 시 "문서를 준비 중입니다" 스낵바 노출. 디바이스 저장소 용량 부족 시 OS 기본 에러 메시지 연동.

## ✅ 4. 수락 기준 (Acceptance Criteria)

> 기능이 완료되었다고 판단하기 위한 최소 조건입니다.

- [ ] 비교 뷰에서 업체 A와 업체 B의 항목이 누락 없이 교차 비교 가능한 UI로 렌더링되어야 한다.
- [ ] PDF 생성 버튼 클릭 시, 폰트 깨짐 현상이나 잘림 현상 없이 앱 내에서 본 예산 명세서와 동일한 데이터가 문서화되어야 한다.
- [ ] 안드로이드와 iOS 양쪽 기기 모두에서 OS의 기본 공유하기(Share Sheet) 기능과 정상적으로 연동되어야 한다.

## ⚙️ 5. 기술적 고려사항 및 제약 (Non-functional / Tech Notes)

- **API 연동:** 프론트엔드 단에서 HTML/Canvas를 PDF로 변환하는 라이브러리(예: html2pdf.js 등) 또는 백엔드 문서 생성 API 활용.
- **성능/보안:** 렌더링 속도 최적화, PDF 파일 내 개인정보 포함 여부 확인.
- **의존성:** 예산 관리(기능 2) 데이터 구조 완성 후 진행 가능.
